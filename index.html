Modul 2: Critical Path Method (CPM)

Manajemen Proyek & Penjadwalan

Peluncuran Aplikasi Pariwisata menggunakan Critical Path Method (CPM) untuk mengidentifikasi jalur kritis waktu penyelesaian proyek.

Parameter Kegiatan

*Rentang batas 8 hingga 20

Data Komponen Jaringan

Panduan Parameter Analisis

Slack / Float (Kelonggaran Waktu): Toleransi keterlambatan (Slack = LS - ES). Jika Slack bernilai 0, maka aktivitas tersebut berada pada Jalur Kritis.
Hitungan Maju (Forward Pass):
• ES (Earliest Start): Waktu paling awal aktivitas dapat dimulai (Maksimum EF pendahulu).
• EF (Earliest Finish): Waktu paling awal aktivitas selesai (EF = ES + Durasi).
Hitungan Mundur (Backward Pass):
• LF (Latest Finish): Waktu paling lambat aktivitas harus selesai (Minimum LS penerus).
• LS (Latest Start): Waktu paling lambat aktivitas harus dimulai (LS = LF - Durasi).
